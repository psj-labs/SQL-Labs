## 1\. 기본 아이디어 정리

우리가 보고 싶은 결과는 두 가지입니다.

-   ① **각 사원의 연봉** (연봉 = `SAL * 12 + NVL(COMM, 0)`)
-   ② **부서별 평균 연봉** (같은 부서 사원들을 묶어서 평균)

즉, 결과 집합에는 “사원 단위 상세 행 + 부서 단위 소계 행”이 함께 있어야 합니다. 이걸 한 번에 만들어 주는 도구가 바로 `ROLLUP`입니다.

## 2\. 정답 쿼리와 컬럼 순서

```
SELECT dno 부서번호,
       eno 사번,
       ROUND(AVG(sal * 12 + NVL(comm, 0))) 년봉
FROM emp
GROUP BY ROLLUP (dno, eno)
ORDER BY dno, eno;
```

### 왜 `ROLLUP(dno, eno)`인가?

`ROLLUP(a, b)`는 항상

-   (a, b) : a와 b로 그룹
-   (a) : a만으로 그룹 (a별 소계)
-   () : 전체 소계 (전체 합계/평균)

따라서 `ROLLUP(dno, eno)`는 다음 계층을 의미합니다:

-   (dno, eno) → 부서별 + 사원별 연봉
-   (dno) → 부서별 평균 연봉
-   () → 전체 평균 연봉

여기서 dno가 먼저 온다는 것은 “부서를 상위 그룹(부모), 사원을 하위 그룹(자식)”으로 잡겠다는 뜻입니다. 그래야 “부서별 평균 연봉”이 자연스럽게 소계로 생성됩니다.

## 3\. 결과 예시로 보는 동작 구조

EMP 테이블이 아래처럼 있다고 가정해 봅시다.

| DNO (부서) | ENO (사번) | SAL | COMM | 연봉(SAL\*12+NVL(COMM,0)) |
| --- | --- | --- | --- | --- |
| 10 | 101 | 3000 | 200 | 36,200 |
| 10 | 102 | 2800 | 100 | 33,700 |
| 20 | 201 | 3500 | 150 | 42,150 |
| 20 | 202 | 3300 | 0 | 39,600 |

`GROUP BY ROLLUP(dno, eno)` 실행 결과 예시:

| DNO | ENO | 년봉(AVG) | 설명 |
| --- | --- | --- | --- |
| 10 | 101 | 36200 | 10번 부서, 101번 사원의 연봉 |
| 10 | 102 | 33700 | 10번 부서, 102번 사원의 연봉 |
| 10 | NULL | 34950 | 10번 부서의 평균 연봉 (부서 소계) |
| 20 | 201 | 42150 | 20번 부서, 201번 사원의 연봉 |
| 20 | 202 | 39600 | 20번 부서, 202번 사원의 연봉 |
| 20 | NULL | 40875 | 20번 부서의 평균 연봉 (부서 소계) |
| NULL | NULL | 37912 | 전체 평균 연봉 (전체 소계) |

※ `ENO`가 `PK`라서 보통 한 행만 존재하므로, 사원별 그룹의 AVG는 실제 연봉과 동일하게 나옵니다. 여러 행이 있는 구조여도 동일한 원리로 동작합니다.

## 4\. 순서를 바꾸면 왜 오답이 되는가? (ROLLUP(eno, dno))

만약 아래처럼 작성하면:

```
GROUP BY ROLLUP(eno, dno)
```

계층 구조는 이렇게 바뀝니다:

-   (eno, dno) : 사원+부서 기준
-   (eno) : 사원별 소계
-   () : 전체 소계

즉, 이 경우에는 **“사원별 요약 → 전체 요약”**만 존재하고, “부서별 평균 연봉” 행이 생성되지 않습니다. 문제 요구사항(부서별 평균 연봉)을 만족하지 못하므로 순서가 잘못된 것입니다.

## 5\. ROLLUP vs CUBE 한 줄 정리

-   `ROLLUP(a, b)`: 계층형 집계. (a, b) → (a) → () 순서. 상위 그룹이 항상 왼쪽부터.
-   `CUBE(a, b)`: 모든 경우의 수 집계. (a, b), (a), (b), () 모두 생성.
-   “부서별 → 사원별 → 전체”와 같이 부모-자식 구조 소계가 필요하면 ROLLUP, “부서별도 보고, 업무별도 보고, 전체도 보고” 같이 모든 축 조합이 필요하면 CUBE.

### 추가: GROUPING()으로 합계 행 구분하기심화

```
SELECT dno,
       eno,
       ROUND(AVG(sal * 12 + NVL(comm, 0))) 년봉,
       GROUPING(dno)  AS grp_dno,
       GROUPING(eno)  AS grp_eno
FROM emp
GROUP BY ROLLUP(dno, eno)
ORDER BY dno, eno;
```

-   `GROUPING(컬럼) = 0` → 실제 그룹 컬럼이 사용된 행
-   `GROUPING(컬럼) = 1` → ROLLUP/CUBE가 만든 소계/총계 행
